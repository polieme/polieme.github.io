  2. 别名上添加双引号，查询出来的就是小写了

MyBatis saveOrUpdate写法

如果按照常规的套路，肯定是现根据ID查询下是否存在，然后如果存在，则走更新方法，如果不存在，走新增方法，但是这种写法真的感觉比较麻烦

selectKey实现saveOrUpdate

  1. 首先使用selectKey标签查询是否存在满足条件的数据，如果有那么count会大于0，没有，count为0
  2. 如果大于0，那么走UPDATE语句，等于0，走INSERT语句

ON DUPLICATE KEY UPDATE实现saveOrUpdate

说明：ON DUPLICATE KEY UPDATE 个人理解是当主键或唯一索引发生重复的时候，走更新SQL，如果没有重复，则走新增SQL，另外还需要说明以下几点：

  • 第一行中的keyProperty是用来指定自动生成的id的字段对应Bean中属性名，这个是用作后续代码使用新增时的ID的时候使用的（像主从表，从表插入时需要主表的主键ID作为从表的外键），在执行完插入之后能直接在传入的parameterType对象中找到新生成的ID
  • 在insert SQL中需要把主键也放进去，也就是这里的gift_id（及时Bean对象中giftId为null，也得放），因为如果不放的话，那会出现一直插入的现象
  • 在这里insert不需要使用autoGenerateKey属性，如果使用了，会出现走updateSQL也会更新主键的情况，这样的话就很头疼，你在进入Mapper之前得先记下之前的ID，所以是画蛇添足，直接不加即可
  • ON DUPLICATE KEY UPDATE后面直接是数据库字段=Bean属性不需要where条件，因为都已经DUPLICATE了，那肯定知道ID是啥了，自己就关联了，表名也是不需要的
